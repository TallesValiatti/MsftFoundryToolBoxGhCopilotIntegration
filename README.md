# Vue.js To-Do List — Static Site on Azure Blob Storage

A **Vue 3** single-page application built with **Vite** that implements a to-do list, bundled and deployed as a static website to **Azure Blob Storage**.

This project was generated by **GitHub Copilot** in Agent Mode using information retrieved from a **Microsoft Foundry Toolbox** via the **Model Context Protocol (MCP)**. The Toolbox exposed a File Search tool over a PDF document describing how to host Vue, React, and Angular apps on Azure Blob Storage. Copilot used that knowledge to scaffold the app and write the deploy script without any manual instructions about how the app should look or work.

## What's inside
* `src/App.vue`: root component containing the full to-do list UI and logic (add, complete, remove, clear completed)
* `src/main.js`: application entry point, mounts the Vue app
* `src/style.css`: global base styles (dark background, reset)
* `vite.config.js`: Vite configuration with the Vue plugin
* `package.json`: project metadata, Vue 3 dependency, and npm scripts (`dev`, `build`, `preview`)
* `deploy.sh`: Bash script that provisions Azure infrastructure and deploys the production build to Blob Storage

## Prerequisites
* **Node.js** 18 or later
* **Azure CLI** installed and available on `PATH`
* An active **Azure subscription**

## Run

Install dependencies and start the development server:

```bash
npm install
npm run dev
```

The app is available at `http://localhost:5173`.

To build for production:

```bash
npm run build
```

Output is written to `dist/`.

## Deploy to Azure Blob Storage

The `deploy.sh` script provisions all required Azure resources from scratch and uploads the production build. It creates a new **resource group** (`rg-todo-app`) and a new **Azure Storage Account** (`StorageV2`, `Standard_LRS`) with a random suffix to ensure a globally unique name. It then enables Static Website Hosting, builds the app, and uploads the `dist/` directory to the `$web` container.

```bash
chmod +x deploy.sh
./deploy.sh
```

The script will open a browser for `az login`. After authentication it runs without further interaction and prints the public URL at the end:

```
==> Deploy complete! Your site is available at:
https://<storage-account-name>.z<region>.web.core.windows.net/
```

## Customizing

To change the target Azure region or resource group name, edit the variables at the top of `deploy.sh`:

```bash
RESOURCE_GROUP_NAME="rg-todo-app"
LOCATION="eastus"
```
